A Ruby Enumerable Method
Or thinking about group_by...

I have talked about Ruby Collections such as Arrays and Hashes before. Arrays and Hashes are both collections that are enumerable, meaning that you can get each of the collection items, one after the other. Basically, they have an each method! So enumerable collections share common characteristics. Wouldn't it be nice to have methods that could be applied to all enumerable collections even your own? Here comes the Ruby Enumerable Module with plenty of powerful methods to use with any enumerable collection! We'll talk about one of them: group_by.

The group_by method breaks a collection into groups based on some given criteria. Let's imagine that I have a collection of first names:

first_names = ["Brian", "David", "Richard", "Sebastien", "Charles", "James"]

To group them by length, I can use the group_by method as follows:

first_names.group_by {|first_name| first_name.length}

In the code above, each first name is passed to the block. And the blockâ€™s result is the length of the first name. group_by returns a hash where each block's result is a key, and each value is an array comprised of the collection items that gave this same result:

=> {5 => ["Brian", "David", "James"], 7 => ["Richard", "Charles"], 9 => ["Sebastien"]}

That's pretty neat, isn't it? So, next time you think about using the each method to do something with a collection, you may want to first check the Ruby Enumerable Module for an easier way out!
