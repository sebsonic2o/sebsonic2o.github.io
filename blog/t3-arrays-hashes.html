Ruby Lists
Or thinking about Arrays and Hashes...

We all make lists of things: Things to do, things we need, things we love, and so on. And we may not want to forget about them. In Ruby, you can use Arrays and Hashes to store this information.

I like to remember dishes I enjoy most, from any of my favorite restaurants. For Indian, it is Shezan and my preferred combination is:

Vegetable Samosa
Chicken Tikka Masala
Garlic Naan
Basmati Rice

In Ruby, I could use the Array structure to store my list as follows:

favorite_shezan_dishes = ["Vegetable Samosa", "Chicken Tikka Masala", "Garlic Naan", "Basmati Rice"]

And we can retrieve any item in the list based on its position counting from 0, not 1. In Ruby talk, it is called the index.

favorite_shezan_dishes[0] would return "Vegetable Samosa"
favorite_shezan_dishes[1] would return "Chicken Tikka Masala"
favorite_shezan_dishes[2] would return "Garlic Naan"
favorite_shezan_dishes[3] would return "Basmati Rice"

In Ruby, the Hash structure is very similar to the Array structure. The difference is that you can label each of the items in the list. In Ruby talk, it is called the key. I can now store my list as follows:

favorite_shezan_dishes = {
appetizer: "Vegetable Samosa",
main: "Chicken Tikka Masala",
bread: "Garlic Naan",
side: "Basmati Rice"
}

And we can retrieve any item in the list based on its key.

favorite_shezan_dishes[:appetizer] would return "Vegetable Samosa"
favorite_shezan_dishes[:main] would return "Chicken Tikka Masala"
favorite_shezan_dishes[:bread] would return "Garlic Naan"
favorite_shezan_dishes[:side] would return "Basmati Rice"

So you would probably use an Array when there is a natural order for your list items. And you would prefer a Hash when a natural label can be associated to your list items.
